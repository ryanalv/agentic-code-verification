# Documentação Técnica do Projeto RAG-MPMG

## Visão Geral do Projeto

O projeto RAG-MPMG (MP-IA) é um pipeline de Recuperação de Resposta Aumentada (RAG) desenvolvido para as normas internas do
Ministério Público de Minas Gerais (MPMG). O sistema oferece uma API para o processamento de documentos e busca vetorial,
transformando documentos normativos (PDF/DOCX) em conhecimento consultável com filtros normativos. Principais funcionalidades
incluem OCR de documentos, extração de metadados, e busca híbrida.

- **Ray**: 2.47.1

- **SQL Server**: vária conforme instalação local

Outras dependências incluem `requests`, `pydantic`, `redis`, entre outros, conforme detalhado em `requirements.txt`.

## Arquitetura do Sistema

O sistema é organizado em uma arquitetura modular dividida nas seguintes componentes principais:
- **app/**: Contém a API, camadas de serviço, repositórios e esquemas de dados.  
- **RAG/**: Inclui a implementação do pipeline RAG para processamento e análise de documentos.  
- **scripts/**: Scripts utilitários para operação e configuração do sistema.  
- **static/**: Arquivos estáticos e templates utilizados pela API.  

Um diagrama de arquitetura básico pode ser representado com o Mermaid:
```
graph TD;
    A[API: FastAPI] --> B[app/]
    B --> C[services/]
    B --> D[repositories/]
    B --> E[schemas/]
    A --> F[RAG Pipeline]
    F --> G[Processamento de Documentos]
```

## Fluxo de Execução

O fluxo principal de execução do projeto inicia com a API FastAPI que gerencia as requisições. O processo pode ser descrito da
seguinte maneira:
1. **Entrada**: O usuário envia um pedido através da API (e.g., uma requisição POST para `/v1/gerar-modelo/{id}`).  
2. **Lógica**: O sistema processa os documentos, executando OCR, extraindo texto e metadados, e vetorizando os dados.  
3. **Saída**: Respostas geradas são retornadas ao usuário ou armazenadas em um banco de dados.  

## Componentes Chave

1. **app/api**: Define as rotas e endpoints da API.  
2. **app/services**: Contém a lógica de negócio e manipulação de dados.  
3. **RAG/**: Implementação do pipeline de RAG, incluindo procesamiento, consulta e resposta.  
4. **app/repositories**: Manipula a interação com o banco de dados (MongoDB, SQL, etc.).  
5. **app/schemas**: Define estruturas e validações de dados utilizando Pydantic.
