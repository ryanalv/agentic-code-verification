# Documentação Técnica Detalhada do Projeto RAG-MPMG

## Visão Geral

O projeto RAG-MPMG representa um esforço revolucionário para integrar técnicas avançadas de inteligência artificial e processamento de dados no contexto jurídico. O objetivo é desenvolver um sistema capaz de recuperar informações de documentos jurídicos de forma automática e eficiente, utilizando a técnica conhecida como RAG (Retrieval-Augmented Generation). Este sistema inovador concentra-se em transformar normas complexas e documentos internos em conhecimentos acessíveis e pesquisáveis, proporcionando aos usuários um acesso sem precedentes a dados que antes eram difíceis de explorar.

Central ao funcionamento do RAG-MPMG está a ideia de simplificar e automatizar o acesso a documentos jurídicos, como normas e doutrina, processando uma variedade de formatos de arquivo e aplicando técnicas avançadas de reconhecimento óptico de caracteres (OCR). A arquitetura do sistema foi meticulosamente planejada para garantir escalabilidade, eficiência e precisão em cada etapa do pipeline de dados, desde a coleta até a recuperação e síntese da informação.

## Tecnologias Utilizadas

O sucesso do RAG-MPMG se sustenta em uma base tecnológica robusta e cuidadosamente selecionada:

- **FastAPI**: Utilizado para o desenvolvimento de APIs REST robustas e escaláveis, permitindo a criação de rotas de API que são tanto eficientes quanto fáceis de manter. FastAPI aproveita as capacidades do Python para tipagem dinâmica com o uso do Pydantic para validação de dados.
  
- **Ray**: Fundamental para permitir o processamento paralelo e distribuído de tarefas, garantindo que o sistema possa processar grandes volumes de dados simultaneamente. Ray é crucial para a parte do pipeline que demanda computação intensiva.
  
- **PyMuPDF e Surya OCR**: Bibliotecas dedicadas à manipulação e interpretação de arquivos PDF e a realização de OCR. Estas ferramentas permitem a extração precisa e rápida de texto de documentos digitalizados, um componente essencial para a normalização de dados.
  
- **FAISS**: Uma poderosa ferramenta para indexação e busca de dados de alta dimensionalidade. Utilizada para implementar o mecanismo de busca vetorial, que sustenta a recuperação rápida de informações.
  
- **MongoDB e SQL Server**: Bancos de dados escolhidos para o armazenamento redundante e eficiente de dados estruturados e não-estruturados, proporcionando consulta rápida e confiável dos dados mantidos no sistema.
  
- **HDFS**: O sistema de arquivos distribuído Hadoop é escolhido para armazenar grandes conjuntos de dados de forma eficiente, crucial para operações intensivas de I/O.

## Arquitetura

### Estrutura do Projeto

Para garantir manutenção fácil e uma separação clara de responsabilidades, o projeto RAG-MPMG é organizado em múltiplos diretórios e módulos:

- **app/**: Este diretório contém toda a lógica de aplicação, abrigando subdiretórios e módulos que tratam da API, núcleo (core), repositórios de dados, esquemas, serviços e casos de uso específicos.
  
- **RAG/**: A implementação central do pipeline RAG reside neste diretório, que abriga todos os passos necessários para a transformação de dados brutos em respostas refinadas.
  
- **static/**: Destinado ao armazenamento de arquivos estáticos necessários durante o processo de execução do sistema.
  
- **main.py**: O ponto de entrada da aplicação. Ele integra e inicia todos os componentes da aplicação, configurando rotas de API e serviços auxiliares necessários ao funcionamento da mesma.
  
- **requirements.txt**: Arquivo que lista todas as dependências do projeto, essenciais para recriar o ambiente necessário ao funcionamento da aplicação.

### Processo do Pipeline RAG

O pipeline do RAG-MPMG é uma sequência de processos altamente coordenados, cada um desempenhando um papel crucial na análise de dados:

1. **Coleta e extração de texto**: Esta fase inicial é crítica, empregando tecnologias como OCR para converter PDFs e outros documentos de texto escaneados em dados passíveis de manipulação.
  
2. **Segmentação e enriquecimento**: Nesta etapa, o conteúdo textual é analisado e enriquecido com metadados, melhorando a estrutura dos documentos e otimizando-os para futuras buscas.
  
3. **Vetorização e indexação**: Os textos processados são vetorizados para permitir comparações de similaridade eficiente, indexados para armazenamento e recuperação rápida.
  
4. **Recuperação e síntese**: Baseando-se em consultas específicas dos usuários, o sistema recupera dados pertinentes e gera respostas sintéticas, oferecendo resumos adequados e direcionados.

## Fluxos de Dados

O projeto RAG-MPMG inclui um intricado fluxo de dados que coordena diversos componentes desde a ingestão até a entrega:

1. **Coleta**: Através do uso de OCR, os documentos são adquiridos e convertidos em formato textual.
2. **Enriquecimento**: Os documentos textuais são enriquecidos com metadados úteis, facilitando futuras buscas e recuperações.
3. **Vetorização**: Este texto enriquecido é então transformado em vetores de alta dimensionalidade, utilizando técnicas de Processamento de Linguagem Natural (PLN).
4. **Busca e Síntese**: Por fim, usuários interagem com o sistema através de consultas de API para buscar informações, resultando na recuperação de dados sintetizados e relevantes, que são apresentados de forma clara e acionável.
