# Documentação Técnica Unificada do Projeto RAG-MPMG

---

## 1. Visão Geral

O projeto **RAG-MPMG** (Retrieval-Augmented Generation - Ministério Público de Minas Gerais) é uma solução de automação jurídica baseada em inteligência artificial, voltada para o processamento massivo de documentos, geração de minutas, extração de informações via OCR, e gerenciamento de tarefas assíncronas com priorização. O sistema integra múltiplos bancos de dados (relacionais, NoSQL, vetoriais, grafos), provedores de IA generativa (OpenAI, Azure OpenAI, Google Gemini), e oferece API robusta baseada em FastAPI. O objetivo é acelerar fluxos judiciais, garantir auditabilidade, resiliência e extensibilidade, permitindo que o MPMG evolua com novas demandas tecnológicas.

---

## 2. Contexto Teórico da Arquitetura

### 2.1. Retrieval-Augmented Generation (RAG)

**Teoria:**  
RAG é um padrão de arquitetura para sistemas de IA que combinam busca de informações em bases de dados (retrieval) com geração de texto ou respostas via modelos generativos (generation). O modelo busca documentos relevantes e os usa como contexto para gerar respostas mais precisas e fundamentadas.

**Aplicação no Projeto:**  
O RAG-MPMG utiliza bancos vetoriais (Qdrant) para busca semântica de documentos, que são então usados como contexto para geração de minutas jurídicas via OpenAI, Azure OpenAI ou Gemini. O pipeline integra busca, montagem de prompt e geração, garantindo precisão e fundamentação.

### 2.2. Bancos Vetoriais (Vector Stores)

**Teoria:**  
Bancos vetoriais armazenam representações matemáticas (vetores) de textos/documentos, permitindo busca por similaridade semântica, essencial para sistemas RAG.

**Aplicação no Projeto:**  
Qdrant é utilizado como backend para armazenamento e busca de embeddings de documentos. O serviço de vector store permite upload, associação e busca de arquivos por similaridade.

### 2.3. Orquestração de Tarefas (Ray + Redis)

**Teoria:**  
Ray é um framework para execução distribuída e paralela de tarefas Python, enquanto Redis é um backend de filas em memória, rápido e confiável.

**Aplicação no Projeto:**  
O QueueManager abstrai o gerenciamento de filas (prioridade, status, métricas) sobre Redis. Ray executa workers que consomem tarefas da fila, processando OCR, geração de minutas, etc.

### 2.4. API RESTful (FastAPI)

**Teoria:**  
FastAPI é um framework moderno para construção de APIs, com validação automática de dados, documentação embutida, e alta performance.

**Aplicação no Projeto:**  
Toda a interface do sistema é exposta via FastAPI, com validação de entrada, padronização de respostas, e documentação acessível (Swagger/OpenAPI).

### 2.5. Arquitetura Modular, Clean e Extensível

**Teoria:**  
Separação clara de responsabilidades, modularidade, desacoplamento, e interfaces bem definidas são pilares para sistemas escaláveis e fáceis de manter.

**Aplicação no Projeto:**  
O RAG-MPMG organiza o código em módulos de API, core, repositórios, schemas, services, com cada camada dedicada a uma responsabilidade específica e fácil de evoluir.

## 8. Fluxos de Dados

### 8.1. Geração/Modificação de Minuta

1. API recebe requisição.
2. Valida parâmetros.
3. Enfileira tarefa via QueueManager.
4. Salva status em Redis.
5. Worker Ray consome tarefa:
   - Busca documentos/modelos.
   - Monta prompt via PromptBuilder.
   - Chama IA.
   - Atualiza documento.
6. Status consultável via API.
7. Logs gravados.

### 8.2. OCR de Documentos

1. API recebe requisição.
2. Enfileira tarefa OCR.
3. Worker consome:
   - Busca documentos.
   - Executa OCR.
   - Atualiza status/resultados.

### 8.3. Gerenciamento de Filas

- Enfileira (pending).
- Desenfileira (processing).
- Conclui (completed).
- Recupera jobs travados.
- Métricas detalhadas.

### 8.4. Fallback de IA

- Se OpenAI/Azure falhar ou lock ativo, fallback para Gemini.
- Lock gerenciado via Redis.

## 9. Pontos de Atenção e Decisões de Design

- **Padronização de Respostas**: Uso de `StandardResponse`.
- **Prioridade de Filas**: Múltiplas filas para balanceamento.
- **Status de Jobs**: Rastreabilidade (pending, processing, completed).
- **Fallback de IA**: Resiliência.
- **Prompt Modular**: Facilidade de manutenção.
- **Segurança**: Volumes, healthchecks, variáveis de ambiente.
- **Extensibilidade**: Modularidade para novas tarefas e provedores.

## 12. Mensagem Didática Final

> **Para não-técnicos:** Imagine o RAG-MPMG como um grande escritório digital, onde cada tarefa (gerar minuta, extrair texto, analisar documento) é enviada para uma fila. Os "robôs" (workers Ray) pegam essas tarefas, buscam informações nos arquivos e bancos, consultam "especialistas" (IA), escrevem minutas ou extraem textos, e registram tudo para acompanhamento. Se um especialista está ocupado, outro assume, garantindo que o trabalho nunca pare. O sistema cuida da ordem, prioridade e rastreabilidade, como um gerente eficiente e confiável.
