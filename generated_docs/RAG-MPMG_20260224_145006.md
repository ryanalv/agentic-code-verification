# Documentação Técnica Detalhada do Projeto RAG-MPMG

## Visão Geral

O projeto RAG-MPMG (MP-IA) é uma API avançada destinada ao processamento de documentos e busca vetorial, com o objetivo de transformar normas internas em conhecimento acessível e consultável. Utilizando tecnologias de ponta, como FastAPI para a construção da API, Ray para processamento distribuído, e uma série de bibliotecas para OCR e manipulação de documentos, o sistema visa a extração, normalização e indexação de documentos legais, facilitando a busca e recuperação de informações relevantes.

## Tecnologias Utilizadas

O projeto RAG-MPMG emprega uma variedade de tecnologias modernas, cada uma desempenhando um papel crucial na funcionalidade do sistema:

- **FastAPI**: Um framework de alto desempenho para a construção de APIs RESTful, conhecido por sua eficiência e simplicidade na definição de endpoints e documentação automática.
- **Ray**: Uma biblioteca para processamento distribuído, que permite a execução de tarefas em paralelo, aumentando a eficiência e a escalabilidade do sistema.
- **Surya OCR**: Uma biblioteca especializada em reconhecimento óptico de caracteres, essencial para a extração de texto de documentos digitalizados.
- **FAISS**: Uma biblioteca de código aberto para indexação e busca vetorial, utilizada para realizar buscas rápidas e eficientes em grandes volumes de dados.
- **MongoDB**: Um banco de dados NoSQL utilizado para o armazenamento flexível de documentos e metadados, permitindo consultas rápidas e escaláveis.
- **SQL Server**: Um banco de dados relacional utilizado para consultas específicas que requerem integridade transacional.
- **HDFS**: Um sistema de arquivos distribuído projetado para armazenar grandes volumes de dados de forma redundante e eficiente.

## Arquitetura do Sistema

A arquitetura do projeto é modular e bem organizada, dividida em várias camadas e componentes que facilitam a manutenção e a escalabilidade:

1. **API**: Implementada com FastAPI, expõe endpoints para interação com o sistema.
2. **Core**: Contém a configuração do sistema, gerenciamento de filas e logging.
3. **Services**: Implementa a lógica de negócios, incluindo processamento de documentos e interação com serviços externos.
4. **Repositories**: Responsável pela interação com bancos de dados, tanto NoSQL quanto SQL.
5. **Schemas**: Define os modelos de dados utilizados nas requisições e respostas da API.

## Contexto Teórico da Arquitetura

### FastAPI

FastAPI é um framework moderno para a construção de APIs RESTful em Python. Ele é conhecido por sua alta performance, que é comparável a Node.js e Go, graças ao uso de ASGI (Asynchronous Server Gateway Interface) e Pydantic para validação de dados. FastAPI também oferece documentação automática de APIs via Swagger e ReDoc, o que facilita a interação e compreensão dos endpoints disponíveis.

### Ray

Ray é uma biblioteca de código aberto para computação distribuída. Ele permite que desenvolvedores escrevam aplicações paralelas e distribuídas em Python com facilidade. Ray é especialmente útil em cenários de machine learning e processamento de dados em larga escala, onde a distribuição de tarefas pode melhorar significativamente o desempenho.

### Surya OCR

Surya OCR é uma biblioteca especializada em reconhecimento óptico de caracteres. Ela é utilizada para converter imagens de texto em texto editável, um processo essencial para a digitalização de documentos físicos. Surya OCR é conhecida por sua precisão e capacidade de lidar com uma variedade de formatos de documentos.

### FAISS

FAISS (Facebook AI Similarity Search) é uma biblioteca de código aberto desenvolvida pelo Facebook AI Research. Ela é projetada para realizar busca de similaridade em grandes coleções de vetores. FAISS é altamente eficiente e pode ser usada para tarefas como busca de imagens, recuperação de informações e recomendação de produtos.

### MongoDB

MongoDB é um banco de dados NoSQL que oferece flexibilidade e escalabilidade. Ele armazena dados em formato de documento (JSON-like), o que permite uma modelagem de dados mais natural e intuitiva. MongoDB é ideal para aplicações que requerem alta disponibilidade e escalabilidade horizontal.

### SQL Server

SQL Server é um sistema de gerenciamento de banco de dados relacional desenvolvido pela Microsoft. Ele é utilizado para armazenar e recuperar dados conforme solicitado por outras aplicações de software. SQL Server oferece suporte a transações ACID, o que garante a integridade dos dados em operações complexas.

### HDFS

HDFS (Hadoop Distributed File System) é um sistema de arquivos distribuído projetado para rodar em hardware commodity. Ele é altamente escalável e pode armazenar grandes volumes de dados, distribuindo-os por múltiplos nós para garantir redundância e alta disponibilidade.

## Análise de Código Detalhada

### `main.py`

O arquivo `main.py` é o ponto de entrada da aplicação. Ele é responsável por carregar as variáveis de ambiente e criar a instância da aplicação FastAPI.

```python
from dotenv import load_dotenv
import ray
import fastapi
from app.api.v1 import router as v1_router
from app.core.config import Config

# Carregar variáveis do .env
load_dotenv()

config = Config()

app = fastapi.FastAPI(
    title="MP-IA",
    version=config.app_version,
    description="API - para processamento de Inteligência Artificial",
    openapi_url="/openapi.json",
    docs_url="/docs",
    redoc_url="/redoc",
)

app.include_router(v1_router)
```

#### Análise

- **Carregamento de Variáveis**: O uso de `load_dotenv()` permite que as variáveis de ambiente sejam carregadas de um arquivo `.env`, facilitando a configuração do ambiente de execução.
- **Configuração da API**: A instância `FastAPI` é configurada com informações como título, versão e URLs para documentação. Isso é fundamental para a usabilidade da API.
- **Inclusão de Roteadores**: O roteador `v1_router` é incluído, o que indica que todas as rotas definidas neste roteador estarão disponíveis na API.

### `app/api/v1.py`

Este arquivo define as rotas da API na versão 1.

#### Principais Endpoints

1. **`GET /`**: Retorna uma mensagem de "Hello World".
2. **`GET /gerar-minuta/{id}`**: Inicia o processamento de um modelo baseado no ID fornecido.
3. **`POST /ocr-documentos-processo`**: Inicia o processamento de OCR para documentos de processo.

#### Análise

- **Gerenciamento de Erros**: Cada endpoint possui tratamento de exceções, registrando erros e retornando mensagens apropriadas ao cliente.
- **Uso de `QueueManager`**: O gerenciamento de tarefas é feito através da classe `QueueManager`, que permite enfileirar e monitorar o status das tarefas.

### `app/core/config.py`

A classe `Config` centraliza a configuração do sistema, carregando variáveis de ambiente e definindo parâmetros de conexão com bancos de dados e serviços.

#### Análise

- **Carregamento de Variáveis**: O uso de `os.getenv` permite que a configuração seja flexível e adaptável a diferentes ambientes.
- **Métodos de Configuração do Ray**: A classe inclui métodos para configurar o ambiente de execução do Ray, o que é crucial para o processamento distribuído.

### `app/core/logger.py`

A classe `Logger` é responsável pela configuração do logging da aplicação.

#### Análise

- **Rotação de Logs**: O uso de `RotatingFileHandler` garante que os logs não cresçam indefinidamente, mantendo um histórico controlado.
- **Níveis de Log**: A configuração de níveis de log permite que a aplicação registre informações relevantes para depuração e monitoramento.

### `app/core/queue.py`

A classe `QueueManager` gerencia as filas de tarefas utilizando Redis.

#### Análise

- **Gerenciamento de Status de Jobs**: A classe mantém o status de cada job (pendente, em processamento, concluído) e permite a consulta e atualização desses status.
- **Enfileiramento de Tarefas**: O método `enqueue_task` adiciona novas tarefas à fila, gerando IDs únicos e salvando o status inicial.

### `app/services/ocr_service.py`

Este arquivo contém funções para realizar OCR em diferentes tipos de documentos.

#### Análise

- **Suporte a Múltiplos Formatos**: As funções `ocr_documento_pdf`, `ocr_documento_html`, e `ocr_documento_docx` permitem a extração de texto de documentos em formatos variados, utilizando bibliotecas específicas para cada tipo.
- **Uso de `Surya`**: A integração com a biblioteca Surya para OCR é um ponto forte, pois oferece um reconhecimento de texto robusto.

### `app/usecases/inteligencia_usecase.py`

Este arquivo contém a lógica de negócios para o processamento de documentos de inteligência.

#### Análise

- **Processamento de Documentos**: Funções como `obter_texto_pdf` e `obter_texto_documento` são responsáveis por extrair texto de documentos e salvar os resultados no banco de dados.
- **Validação de OCR**: A função `validar_ocrs` assegura que a taxa de falhas no OCR não exceda um limite aceitável, garantindo a qualidade do processamento.

## Fluxos de Dados
