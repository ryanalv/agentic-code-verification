# Documentação Técnica do Projeto RAG-MPMG

## Visão Geral

O projeto RAG-MPMG é uma aplicação desenvolvida para implementar um pipeline de Recuperação de Informação, Análise e Geração (RAG) aplicado às normas internas do Ministério Público de Minas Gerais (MPMG). A aplicação é construída em Python e utiliza uma API RESTful para processamento de documentos e busca vetorial. O sistema é projetado para transformar documentos normativos em conhecimento consultável, utilizando técnicas de OCR (Reconhecimento Óptico de Caracteres) e vetorização.

## Tecnologias

O projeto incorpora uma variedade de tecnologias e bibliotecas, incluindo:

- **FastAPI**: Framework para construção de APIs web de forma rápida e eficiente.
- **Ray**: Biblioteca para execução de tarefas em paralelo e processamento distribuído.
- **Surya OCR**: Ferramenta para reconhecimento de texto em documentos.
- **FAISS**: Biblioteca para busca vetorial e indexação de dados.
- **MongoDB**: Banco de dados NoSQL utilizado para armazenar documentos e metadados.
- **SQL Server**: Sistema de gerenciamento de banco de dados relacional.
- **HDFS**: Sistema de arquivos distribuído utilizado para armazenamento de grandes volumes de dados.

## Arquitetura

A arquitetura do projeto é modular, dividida em várias camadas que incluem:

1. **API**: Camada que expõe os endpoints da aplicação, permitindo a interação com o sistema.
2. **Core**: Contém a configuração, log, gerenciamento de filas e utilitários.
3. **Repositories**: Responsável pela interação com os bancos de dados (MongoDB e SQL Server).
4. **Schemas**: Define os modelos de dados utilizados na aplicação.
5. **Services**: Contém a lógica de negócio e serviços que realizam operações específicas, como OCR e manipulação de vetores.
6. **Use Cases**: Implementa os casos de uso da aplicação, orquestrando as interações entre as diferentes camadas.

## Contexto Teórico da Arquitetura

### FastAPI

FastAPI é um framework moderno para a construção de APIs em Python, conhecido por sua alta performance, suporte a validação de dados e geração automática de documentação. Ele utiliza o padrão de programação assíncrona, permitindo que operações de I/O sejam realizadas de forma eficiente.

### Ray

Ray é uma biblioteca de código aberto para computação distribuída. Ele permite que desenvolvedores escalem aplicações Python de forma simples, utilizando paralelismo em múltiplos nós. No contexto do RAG-MPMG, Ray é utilizado para distribuir tarefas de processamento de documentos.

### OCR (Reconhecimento Óptico de Caracteres)

OCR é uma tecnologia que converte diferentes tipos de documentos, como PDFs ou imagens capturadas por câmera, em dados editáveis e pesquisáveis. No projeto, o Surya OCR é utilizado para extrair texto de documentos normativos.

### FAISS

FAISS (Facebook AI Similarity Search) é uma biblioteca para busca vetorial eficiente. Ela é usada para encontrar vetores semelhantes em grandes conjuntos de dados, sendo ideal para aplicações de busca de similaridade.

### MongoDB e SQL Server

MongoDB é um banco de dados NoSQL que armazena dados em formato de documento, enquanto o SQL Server é um banco de dados relacional. Ambos são utilizados no projeto para armazenar diferentes tipos de dados, como documentos e metadados.

### HDFS

HDFS (Hadoop Distributed File System) é um sistema de arquivos distribuído projetado para armazenar grandes volumes de dados em clusters de computadores. Ele é utilizado no projeto para armazenar documentos normativos em larga escala.

## Fluxos de Dados

O fluxo de dados no sistema pode ser descrito da seguinte forma:

1. **Recepção de Solicitações**: A API recebe solicitações através de endpoints definidos.
2. **Processamento de Tarefas**: As solicitações são enfileiradas utilizando o `QueueManager`, que gerencia as tarefas em Redis.
3. **Execução de OCR**: As tarefas de OCR são processadas, extraindo texto de documentos e armazenando os resultados.
4. **Retorno de Respostas**: As respostas são formatadas utilizando o modelo `StandardResponse` e enviadas de volta ao cliente.
